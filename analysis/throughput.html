FP32 Throughput Comparison


We show the throughput results on different GPU architectures including the latest Pascal & Volta generations. Unlike the baseline analysis on P4000, we use a fixed mini-batch size for each benchmark. All benchmarks are trained in single-precision floats.


As we can see that for image classification benchmarks, V100-SXM2 outperforms all other GPU architectures. The second tier includes the V100-PCIe, 2080 Ti and Titan V. Their performance is roughly 15% slower than V100-SXM2. The Pascal architectures deliver similar performance comparing to 1080 Ti, which is slightly (10%) less than Titan Xp. We observed similar performance tiers in Transformer and Faster RCNN benchmarks.


For the transformer model, the GPU architectures below the second tier (V100-PCIe, 2080 Ti, Titan V) are much slower than (roughly 50% slower) others. For Faster RCNN they are also around 30% slower, similar to the image classification models as Faster RCNN essentially use an image classification network as its base model.

For the seq2seq model which consists of mainly RNN layers, implemented by MXNet sockeye, V100-SXM2 still delivers the best training speed, but the V100-PCIe architecture is much slower and only better than P100. The Titan Xp and 1080 Ti architectures appear to deliver better performance results.


However for the NMT benchmark, the V100-PCIe is slightly faster than V100-SXM2, as well as Titan V and Titan Xp.


The A3C model uses a relatively shallow CNN network as its base model. In this case the V100 architectures are not able to outperform other architectures.
